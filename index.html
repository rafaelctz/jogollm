O que é Machine Learning?

No módulo anterior aprendemos que um algoritmo é uma lista de instruções que nós escrevemos. O programador diz exatamente o que o computador deve fazer em cada situação.

Machine Learning (Aprendizado de Máquina) é diferente: o computador aprende a partir de exemplos, sem que ninguém precise escrever todas as regras. Ele encontra os padrões sozinho!

🧠 Machine Learning = dar muitos exemplos ao computador para que ele descubra as regras por conta própria

🆚 Algoritmo tradicional vs. Machine Learning

Veja a diferença na prática:

📋

Algoritmo Tradicional

O programador escreve cada regra manualmente:

"SE o e-mail tem a palavra 'promoção' E vem de remetente desconhecido → ENTÃO é spam"

Regras fixas. Quem errar a regra, o sistema erra também.

🧠

Machine Learning

O sistema analisa 10.000 e-mails spam e 10.000 e-mails normais.

Ele mesmo descobre quais padrões indicam spam — palavras, hora de envio, frequência — sem que ninguém explique as regras.

💡 Pense assim: no algoritmo, você ensina o peixe a pescar explicando cada movimento. No ML, você mostra mil fotos de pesca e o computador aprende sozinho como fazer.

🎓 Os 3 tipos de aprendizado

👩‍🏫

Supervisionado

Aprende com exemplos rotulados. "Esta foto é um gato ✓. Esta foto é um cachorro ✓." A máquina recebe a resposta certa durante o treino.

🔍

Não-supervisionado

Aprende sem rótulos — agrupa dados parecidos por conta própria. Como separar alunos em grupos por perfil de aprendizagem.

🎮

Por reforço

Aprende tentando e errando, recebendo recompensas pelos acertos. Como um jogo: faz algo bom → ganha ponto → repete o comportamento.

👶 A Máquina Aprende como uma Criança

Uma criança aprende a reconhecer animais vendo muitos exemplos. Uma rede neural faz exatamente isso! Veja como o processo acontece:

📚 Como funciona o treinamento?

1️⃣ Dados de treinamento

Mostramos milhares de exemplos com as respostas corretas: fotos de gatos com o rótulo "gato", fotos de cachorros com o rótulo "cachorro".

2️⃣ O modelo chuta uma resposta

O modelo tenta adivinhar. No começo, ele erra bastante — como uma criança que ainda está aprendendo as palavras.

3️⃣ Calcula o erro

Comparamos a resposta do modelo com a resposta certa. Quanto maior a diferença, maior o erro. Isso chama-se "função de perda".

4️⃣ Ajusta os pesos

O modelo ajusta seus parâmetros internos para errar menos na próxima vez. Como corrigir o lápis na mão depois de escrever feio.

5️⃣ Repete milhares de vezes (épocas)

O ciclo se repete com todos os exemplos — uma época é uma passagem completa pelos dados. Com muitas épocas, o modelo fica cada vez mais preciso.

💡 Como o modelo "decide"?

O modelo aprende quais palavras aparecem mais em textos positivos e negativos. Se você escrever algo com "incrível", "excelente", "ótimo" → ele aposta em positivo. Se tiver "péssimo", "horrível", "decepcionante" → negativo. Isso é chamado de Naive Bayes — um dos algoritmos de ML mais simples e poderosos!

🏫 Conexão pedagógica: Imagine um sistema que analisa as respostas escritas dos alunos e automaticamente identifica quem está frustrado, engajado ou confuso — sem o professor ler cada texto. É assim que plataformas adaptativas funcionam!
⚠️ Senso Crítico

O Perigo dos Vieses no Machine Learning

Machine Learning aprende com dados do mundo real — e o mundo real tem desigualdades. Se os dados de treinamento forem enviesados, o modelo vai aprender e amplificar esses preconceitos.

⚡ Caso real: em 2018, a Amazon abandonou um sistema de ML para contratar funcionários porque ele penalizava currículos de mulheres — porque foi treinado com dados históricos que refletiam uma indústria predominantemente masculina.

🎮 Analise os Cenários

Leia cada situação e identifique o problema. Isso é fundamental para quem vai trabalhar com tecnologia na educação!

🛡️ Como combater os vieses?

📊

Dados Diversos

Incluir exemplos de diferentes grupos, gêneros, etnias, regiões geográficas e contextos socioeconômicos no treinamento.

🔍

Auditorias Regulares

Testar o sistema periodicamente com casos conhecidos para detectar discriminações antes que causem dano.

👥

Equipes Diversas

Incluir pessoas de diferentes perfis nas equipes que desenvolvem os sistemas. Perspectivas diversas identificam problemas que grupos homogêneos não percebem.

🧑‍🏫

Humano no Loop

Decisões importantes — como avaliar alunos ou contratar professores — devem sempre ter um humano revisando as sugestões da IA.
